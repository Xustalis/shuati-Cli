# Shuati CLI 深度改进方案与实施路线图 

本项目旨在为算法竞赛选手提供极致流畅、智能且稳健的本地开发体验。针对近期收集的 Issue 及用户反馈，我们将重点从“功能堆叠”转向“本地体验夯实”，制定以下 v3.0 演进方案。

---

## 1. 深度问题剖析与对策

### 🛠️ 性能与兼容：低端机“零负担”计划
- **问题**：本地 LLM 模型资源占用过高，且易产生误导。
- **对策**：采用 **“分层诊断”**。确定性问题（如语法、环境）由**本地离线规则引擎**处理；策略性建议由云端 AI 按需响应。

### 🧩 体验碎片化：Shell 习惯兼容
- **问题**：REPL 模式下不支持通用 Shell 命令（ls, cd, cls），导致操作断层。
- **对策**：内置指令模拟。让用户像在系统终端一样操作文件，支持 `cd` 后的项目动态重载。

### 🚪 新手门槛：“路径迷失”保护
- **问题**：新手习惯“双击运行”，导致 CWD 错误，程序无法加载。
- **对策**：**智能启动守卫 (Boot Guard)**。若启动位置非法，自动开启 TUI 引导界面（支持一键初始化或历史路径跳转）。

---

## 2. 核心改进模块详解

### ① 编译器“急诊室” (Tier 1: 离线规则引擎)
- **原理**：拦截 GCC/Clang 报错，通过正则建立**中英映射字典**。
- **效果**：将 `error: stray '\357'` 瞬时转译为 `[诊断] 检测到中文符号，请检查标点。`
- **技术实现**：
    1.  **错误捕获**：重定向 stderr，实时解析编译器输出流。
    2.  **规则库**：使用 JSON 格式存储正则规则 (`rules/compiler_errors.json`)。
    3.  **匹配策略**：支持 fallback 机制（精确匹配 → 模糊匹配 → 原始错误）。
    4.  **扩展性**：支持 `shuati add-rule` 自定义规则。

### ② 本地代码“考古”索引
- **功能**：自动扫描用户以往的解决方案，提取算法标签。
- **创新**：在写新题时提供历史逻辑交叉引用（如：“你 3 天前在题号 101 使用过类似的 BFS，是否参考？”）。
> [!WARNING]
> **风险提示**：
> - **性能瓶颈**：大量文件扫描可能耗时，需采用增量索引 + 后台异步构建。
> - **隐私问题**：需明确告知用户并获得授权。

### ③ “编译错误基因”图谱 (Error DNA)
- **功能**：可视化记录用户的错误趋势。
- **价值**：发现弱点（如“经常忘写分号”），辅助用户进行针对性修正。

## 3. 完整更新顺序 (Roadmap)

我们建议按以下四个阶段逐步推进。

### **第一阶段：生存保障 (Priority: P0)**
> **目标**：解决“程序跑不起来”和“报错看不懂”的问题。

**实施项目**：
1.  **Boot Guard 实现**：识别双击启动，引导新手进入工作目录。
2.  **编译错误本地化**：在 `CompilerDoctor` 中加入常用错误规则。
3.  **Windows 右键集成**：添加“在此处打开 Shuati CLI”注册表脚本。

**验收标准**：
- [ ] 双击启动后能识别并提示用户选择工作目录（覆盖率 100%）。
- [ ] 前 20 个常见编译错误能正确中文化（准确率 ≥95%）。
- [ ] Windows 系统右键菜单正常工作。

### **第二阶段：操作可用性 (Priority: P1)**
> **目标**：让 CLI 变得好用、顺手。

**实施项目**：
1.  **REPL 指令扩展**：支持 `ls`, `cd`, `clear`, `pwd`。
2.  **架构解耦**：拆分 `main.cpp` 至 Command 模式。
3.  **异步 IO 引入**：确保扫描文件时不阻塞 UI。

**验收标准**：
- [ ] `cd` 命令能正确切换上下文并刷新题目列表。
- [ ] 长时间 IO 操作界面不卡顿（FPS ≥ 30）。

### **第三阶段：去中心化提效 (Priority: P2)**
> **目标**：降低对 API 的依赖，强化本地能力。

**实施项目**：
1.  **静态检查单 (Heuristic Engine)**：内置各算法分类的 Checklist。
2.  **本地代码读取系统**：实现历史题目的关键词匹配与参考提示。
3.  **测试用例生成器增强**：引入基于范围的随机生成。

### **第四阶段：深度智能与闭环 (Priority: P3)**
> **目标**：打造差异化竞争力，建立反馈闭环。

**实施项目**：
1.  **Error DNA Dashboard**：展示个人错误分布热力图。
2.  **Insta-Feedback**：集成一键反馈 issue 捕获器。
3.  **AI 题目炼金术**：实现变体题目生成逻辑。
